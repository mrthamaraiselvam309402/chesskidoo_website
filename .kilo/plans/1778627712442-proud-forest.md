## Step 4: Arena JS — AI Engine (assets/js/arena.js)
**New file** — ~800 lines, self-contained IIFE extending `CK.arena`

### 4a. Chess Engine (Stockfish WASM)
- Load Stockfish WASM from CDN: `https://cdn.jsdelivr.net/npm/stockfish.js@10.0.2/stockfish.wasm.js`
- Fallback: enhanced minimax (3-ply + alpha-beta + piece-square tables) if WASM fails
- Difficulty levels map to search depth: Beginner=1, Intermediate=2, Advanced=3, Expert=4
- UCI communication via `postMessage`/`onmessage`

### 4b. Board Logic
- `chess.js` for move validation, FEN/PGN handling, game state
- Custom renderer: builds 8x8 grid from FEN, handles drag-and-drop + click-to-move
- Legal move generation per square (highlight dots)
- Move history with PGN export

### 4c. Real-time Analysis
- After each player move, send position to Stockfish for analysis
- Display: evaluation (centipawns → pawns), best line (first 3 moves), depth
- Classification engine: compare player move vs best move
  - **Brilliant** (green): only good move, sacrifices material
  - **Best** (green): matches engine top choice
  - **Excellent** (light green): within 0.2 pawns of best
  - **Good** (blue): within 0.5 pawns
  - **Inaccuracy** (yellow): 0.5–1.0 pawn loss
  - **Mistake** (orange): 1.0–2.0 pawn loss
  - **Blunder** (red): >2.0 pawn loss or missed win

### 4d. Post-Game Report
Auto-triggered on checkmate/stalemate/resign/draw. Contains:
- **Game summary:** result, total moves, opening name (ECO code approximation), duration
- **Accuracy %:** weighted average of move classifications (both sides)
- **Move breakdown:** bar chart (Chart.js) showing brilliant/good/inaccuracy/blunder counts per side
- **Evaluation graph:** line chart of eval over move number (like chess.com)
- **Key moments:** auto-detected blunders/mistakes with move numbers, clickable to review
- **Performance rating estimate:** based on accuracy and difficulty

### 4e. Digital Certificate
- Generated as HTML element (printable)
- Contains: ChessKidoo logo, "Certificate of Achievement", player name, date, difficulty level, result, accuracy %, performance grade (S/A/B/C/D), decorative gold border, unique certificate ID
- "Download as PDF" button using `window.print()` with print-specific CSS
- "Share" button (copies link / uses Web Share API)

### 4f. Game Flow
```
Landing Page → [Test Your Skill btn] → Arena Page
  → Select Difficulty → [Start Game]
  → Player moves → AI thinks (500-2000ms) → AI moves
  → Real-time analysis updates
  → Game ends → [Post-Game Report modal]
  → [View Certificate] → [Certificate modal]
  → [Play Again] or [Back to Home]
```

## Key Technical Decisions

1. **Stockfish WASM over simple minimax**: Provides genuine engine-level analysis for meaningful post-game reports. CDN-hosted, no build step.

2. **Custom board renderer over chessboard.js**: The existing chessboard.js is only used for the hero demo. The arena gets a purpose-built board for full control over analysis overlays, eval bar, and move classification indicators.

3. **Client-side only**: No server needed. All analysis, report generation, and certificate creation happens in the browser.

4. **Chart.js already loaded**: Used for the evaluation graph and accuracy bar chart in the post-game report.

5. **Print-friendly certificate**: Uses `@media print` CSS rather than a PDF library to avoid adding dependencies.

---

## Risks & Mitigations

- **Stockfish WASM load time**: Show loading spinner; fallback to minimax if WASM fails to load within 5s
- **Mobile performance**: Reduce analysis depth on mobile; disable eval graph on small screens
- **Large file sizes**: Stockfish WASM is ~1MB; loaded lazily only when arena page opens
